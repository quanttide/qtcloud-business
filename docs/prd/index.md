# 产品需求文档

量潮商务云（qtcloud-business）产品需求文档，聚焦商务获客与客户管理系统设计。

## 产品定位

商务云是企业在 B2B 销售场景下的获客与转化工具，帮助商务经理降低交易成本、减轻心理压力。

→ [查看 BRD](../brd/index.md)

## 用户故事

### 获客渠道

**US1**: 作为商务经理，我希望将案例信息发布到多个渠道，以便获得更多潜在客户的咨询。

- 接受标准：
  - 能录入案例的标题、客户、行业、痛点、解决方案、效果
  - 能选择发布渠道并一键发布
  - 能查看各渠道的曝光量、咨询量

**US2**: 作为商务经理，我希望案例能自动同步到各个平台，以便节省手动发布的时间。

- 接受标准：
  - 发布后 5 分钟内同步到各平台
  - 同步状态可查看

### 信任背书

**US3**: 作为商务经理，我希望在接触客户时有真实案例可展示，以便建立信任。

- 接受标准：
  - 能按行业/场景搜索案例
  - 能打开案例详情页展示给客户
  - 案例自动脱敏处理

**US4**: 作为商务经理，我希望看到案例被查看后带来的咨询转化，以便评估案例效果。

- 接受标准：
  - 能查看每个案例的曝光量、咨询量、转化数

### 潜客管理

**US5**: 作为商务经理，我希望系统自动归集各渠道的潜客信息，以便集中管理。

- 接受标准：
  - 自动归集公众号留言、官网咨询等渠道的潜客
  - 能查看潜客的来源、需求、联系方式

**US6**: 作为商务经理，我希望记录潜客的跟进情况，以便避免遗漏。

- 接受标准：
  - 能添加潜客沟通记录
  - 能设置下次跟进提醒

### 交易推进

**US7**: 作为商务经理，我希望看到每个客户的交易阶段，以便高效推进。

- 接受标准：
  - 能将潜客标记为不同阶段（需求确认/方案报价/合同签署/交付验收）
  - 能看到所有潜客的阶段分布

**US8**: 作为商务经理，我希望有标准化的销售物料可以使用，以便减少准备时间。

- 接受标准：
  - 能生成公司简介、方案PPT、报价单
  - 能从知识库查找常见问题答案

## 核心指标

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| **案例曝光量** | 所有渠道案例的累计曝光次数 | SUM(各案例 views) |
| **咨询转化率** | 曝光到咨询的转化比例 | COUNT(inquiries) / SUM(views) |
| **签约转化率** | 咨询到签约的转化比例 | COUNT(contracts) / COUNT(inquiries) |
| **平均签单周期** | 从首次接触到签约的平均天数 | AVG(signed_at - first_contact_at) |
| **潜客跟进率** | 潜客在规定时间内被跟进的比例 | COUNT(跟进且在24h内) / TOTAL |

## 技术约束

- **AI 生成延迟**：案例描述生成在 5 秒内完成
- **多渠道同步**：案例发布后 5 分钟内同步到各平台
- **数据安全**：HTTPS/TLS 1.3 传输，敏感信息脱敏处理
- **多端支持**：Web、iOS、Android 三端数据实时同步
